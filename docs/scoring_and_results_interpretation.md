## 一、三种「结果」不要混用

| 层级 | 环境 | 能看到什么 | 是否正式得分 |
|------|------|------------|--------------|
| **A. SCNet 自测** | 容器 `testdata/run_throughput.sh` | 吞吐、TTFT P99、TPOT P99 明细 | **否**（调试对照） |
| **B. 官方平台评测** | pra 提交 → 编译 → `launch.sh` → bench | AC/失败、三档吞吐、最终得分、SLA/精度扣分 | **是** |
| **C. 希冀排行榜** | 平台结果汇总 | 三档吞吐、最终得分、扣分、排名 | **是**（公开摘要） |

**硬规则**：

- 排行榜**不展示** TTFT/TPOT 数值、OpenCompass 明细、编译 log。
- SCNet 数字**不能**外推平台得分（本队 7/10 已验证：SCNet ~13.7 tok/s vs 平台 16–32K **5.77**）。
- 优化决策：**SCNet 看 TTFT/TPOT 与 A/B；平台/排行榜看正式得分。**

## 二、官方评分公式（必记）

### 2.1 总公式

```
最终得分 = 吞吐量得分 × 精度系数

吞吐量得分 = 4–8K 得分(20%) + 8–16K 得分(50%) + 16–32K 得分(30%)
```

### 2.2 单档吞吐得分

```
单档相对提升率 r = (你的吞吐 − 官方 Baseline 吞吐) / 官方 Baseline 吞吐

单档得分 = 档位权重 × [ 60% + 40% × (1 − e^(−1.3 × r)) ]
```

| r | 单档得分占满分比例 | 含义 |
|---|-------------------|------|
| **0**（= Baseline） | **60%** | 没优化也有保底 60% 档分 |
| **0.5**（+50%） | ~79% | 明显提升 |
| **0.74**（+74%） | ~85% | 榜上强队量级（如他队 ~84 分） |
| **< 0**（慢于 Baseline） | **< 60%** | 负优化，严重拖分 |

**特性**：提升率进入收益递减区；**慢于 Baseline 比「没提升」更惨**（指数项 > 1 时档分暴跌）。

### 2.3 SLA 熔断（不单独扣分，但可清零档分）

| 指标 | 统计方式 | 门限 |
|------|----------|------|
| **TTFT P99** | **分档**各自统计 | ≤ 该档 Baseline TTFT P99 × **1.5** |
| **TPOT P99** | **全局**汇总 | ≤ Baseline TPOT P99 × **1.5** |

**任一档位** bench 时 TTFT 或 TPOT 超标 → **该档吞吐得分直接清零**（不是从总分里扣几分）。

排行榜「SLA 扣分 = 0」通常表示**未触发熔断**；不等于 TTFT/TPOT 数值会在榜上展示。

### 2.4 精度系数

四类任务（问答 / 摘要 / 检索 / 聚合）各算精度下降 Δ，查表得 kᵢ，**取平均**为精度系数：

| Δ | k |
|---|---|
| ≤ 1% | **1.00** |
| 1–2% | 0.97 |
| 2–3% | 0.94 |
| … | … |
| > 10% | 0 |

排行榜「精度扣分 = 0」≈ 四类任务均 Δ ≤ 1%，k = 1.0。

### 4.3 本队 7/10 提交解读（59.97）

平台返回：

```
AC; 最终得分=59.9678;
4K-8K实际吞吐=12.92; 8K-16K实际吞吐=10.04; 16K-32K实际吞吐=5.77;
SLA扣分=0.0; 精度扣分=0.0
```

| 档位 | 实测 | vs Baseline | 提升率 r | 含义 |
|------|------|-------------|---------|------|
| 4–8K | 12.92 | +22% | 0.22 | 略高于 Baseline |
| **8–16K** | **10.04** | **+5%** | **0.05** | 几乎等于 Baseline（50% 权重几乎保底） |
| **16–32K** | **5.77** | **−26%** | **−0.26** | **低于 Baseline，负优化** |
| **合计** | | | | **~60 ≈「没优化」档** |

**深度解读**：

1. **AC + SLA=0 + 精度=0** → 能跑、未熔断、精度未掉；低分**纯粹是吞吐接近/局部低于 Baseline**。
2. 59.97 与公式「r≈0 → 约 60 分」一致 → **不是相对某次 84 分倒退，而是本队尚未做出有效加速**。
3. **16–32K 负提升** 嫌疑：gpu=0.95 / 插件入口 / warmup / Graph。
4. 与榜首差距：先把本队做到稳定高于 Baseline，再谈冲 8–16K。

## 五、SCNet 自测数据解读（2026-07-10）

### 5.1 截图摘要

| 指标 | 值 |
|------|-----|
| 输出吞吐 | **13.70 tok/s**（峰值 16.00） |
| TTFT P99 | **4857 ms** |
| TPOT P99 | **69.22 ms** |
| 并发 | 1 |

### 5.2 与平台对照

| 假设档位 | SCNet | 平台 7/10 | 可信度 |
|----------|-------|-----------|--------|
| 16–32K | 13.70 | **5.77** | **不可信外推**（差 2.4×） |
| 8–16K | 13.70 | 10.04 | 若同档，SCNet 仍偏高 37% |

**结论**：SCNet 自测**不能**用来预测平台排名；必须以平台提交为准。

### 5.3 物理意义（仍可用于优化方向）

| 指标 | 解读 | 优化方向 |
|------|------|----------|
| TPOT **69 ms** | 高于 DCU 讲义权重 IO 理论 **~49 ms**（约 +40%） | O2 HIP Graph、O3 GQA（减 launch / KV 读） |
| TPOT P99 极稳 | decode 路径一致，非偶发尖刺 | 系统性带宽/调度问题，非 SLA 偶发 |
| TTFT **4.86 s** | 介于讲义 8K/16K 之间 | O4 warmup/prefix；长档 O(S²) attn |
| 若 TPOT baseline≈49ms | 69/49 ≈ **1.41×** | **< 1.5× 熔断线**，余量约 **6 ms** |

## 七、熔断场景演算（为什么 SLA 比「慢一点点」更可怕）

假设 8–16K 吞吐很高但 TTFT P99 超标：

```
4–8K  得分 ~17
8–16K 得分 0    ← 熔断
16–32K 得分 ~25
吞吐总分 ~42 × 精度系数 → 可能还不如 7/10 的 59.97
```

**教训**：冲吞吐时 **G 必须盯 TTFT/TPOT**；7/10 的问题是**没熔断但吞吐太低**，属于另一类翻车。

## 八、优化优先级（由评分公式反推）

| 优先级 | 动作 | 公式依据 |
|--------|------|----------|
| **P0** | **去掉负优化**（stock 入口 + gpu≤0.94 + 关 warmup + enforce-eager） | 先让三档都 > Baseline，总分明显 >60 |
| **P0** | **修 16–32K 负提升**（禁止默认 0.95） | 负 r 对 30% 权重杀伤大 |
| **P1** | **8–16K 主攻**（50% 权重） | 每 +1 tok/s 约 0.5–1 总分 |
| **P2** | 16–32K 抬回并超过 Baseline | 30% 权重 |
| **P3** | 4–8K 微调 | 20% 权重，ROI 较低 |
| **停止** | 把「富贵花开 84 分」当本队历史；未验证就开 0.95/插件/kernel | 误导决策 |
